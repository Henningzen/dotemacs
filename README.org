** clojure-lsp
   Installed with script in /usr/local/bin,
   https://emacs-lsp.github.io/lsp-mode/tutorials/clojure-guide/  (Eric Dallo)

   Official guide, but with tweaks:

   What I kept:
   + Performance tuning (gc-cons-threshold, read-process-output-max)
   + Hooks for clojure-mode, clojurescript-mode, clojurec-mode
   + LSP as the intelligence layer
     
   What I have changed or removed:
   + company-mode (for now)
   + LSP provides completion via completion-at-point
   + Works with our existing Vertico/Orderless stack
   + Simpler, fewer moving parts
   + Removed flycheck - LSP has built-in diagnostics.
     
   Characteristics (observed so far):
   + lsp-mode shows errors/warnings automatically
   + No need for separate checker
   + Removed lsp-treemacs

   Usage (so far):
   + Jump to definition - M-. (lsp-ui-peek)
   + Find references - M-?
   + Completion - Works with TAB/completion-at-point
   + Rename - C-c l r
   + Code actions - C-c l a
   + Hover documentation - Automatic in minibuffer
   + Diagnostics - Errors/warnings inline
   + Compliant with Cider
   + Paredit and structural editing untouched

* Using Emacs
** General
| Key       | Description                                  |
|-----------+----------------------------------------------|
| ~C-x u~   | Undo tree                                    |
| ~C-k~     | Kill to end of form (structurally safe)      |
| ~M-k~     | Kill whole form                              |
| ~C-M-k~   | Kill forward expression                      |
| ~C-x C-f~ | Open a file. Starts in the current directory |

** Hippie-expand basics
   Usage patterns:
| ~M-/~     | Smart expansion with all strategies                       |
| ~C-M-/~   | Expand full lines only (great for repeating similar code) |
| ~C-c /~   | Case-sensitive expansion                                  |
| ~C-c M-/~ | Words/symbols only (no file paths or lines)               |
| ~C-c C-/~ | Current buffer only (faster, less noise)                  |
| ~C-c f /~ | File name completion only                                 |

** Paredit
   Paredit operations are semantic:
| Key       | Description          |
|-----------+----------------------|
| ~C-→~     | slurp (expand form)  |
| ~C-←~     | barf (contract form) |
| ~M-s~     | splice (unwrap)      |
| ~M-(~     | wrap                 |
| ~C-M-f/b~ | navigate by form     |

** Navigation:
| Key         | Description                   |
|-------------+-------------------------------|
| ~C-M-f~     | Forward by expression         |
| ~C-M-b~     | Backward by expression        |
| ~C-M-u~     | Up in tree                    |
| ~C-M-d~     | Down in tree                  |
| ~C-→~       | Slurp right (pull into form)  |
| ~C-<right>~ | Slurp right (pull into form)  |
| ~C-←~       | Barf right (push out of form) |
| ~C-<left>~  | Barf right (push out of form) |
| ~C-M-←~     | Slurp left                    |
| ~C-M-→~     | Barf left                     |

** Wrapping:
| Key               | Description              |
|-------------------+--------------------------|
| ~M-(~             | Wrap with ()             |
| ~M-s~             | Splice (remove wrapping) |
| ~M-S (shift-s)~   | Split expression         |
| ~M-J~             | Join expressions         |

** Consult
| Key     | Description                                                                  |
|---------+------------------------------------------------------------------------------|
| ~C-x b~ | consult-buffer    - Buffer switching with groups (files, buffers, bookmarks) |
| ~M-s l~ | consult-line      - Jump to any line in current buffer                       |
| ~M-g g~ | consult-goto-line - Better goto-line with preview                            |
| ~M-g i~ | consult-imenu     - Navigate code structure (great for Clojure!)             |
