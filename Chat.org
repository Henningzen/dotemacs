* Company Mode and Your LSP Warning

** What Company Mode Is

Yes, precisely. =company-mode= (COMPlete ANYthing) is the package
that provides those inline dropdown completion menus you see in
YouTube demonstrations—candidates appear as you type, filtered
in real time, with documentation sidebars and the like.

It is, however, /not/ the only way to achieve completion in Emacs.

** Why You See the Warning

Your =setup-lsp.el= correctly sets:

#+begin_src elisp
(setq lsp-completion-provider :capf)
#+end_src

This tells =lsp-mode= to use the built-in =completion-at-point=
(CAPF) framework rather than =company-mode=. However, =lsp-mode=
still /attempts/ to detect and configure =company-mode= at
startup. When it cannot find it installed, it emits that warning.

The warning is cosmetic—your completion pipeline via CAPF +
Vertico + Orderless is intact and functional.


* Two Options to Resolve

** Option A: Suppress the Warning (Recommended)

You already have a coherent completion stack (Vertico, Orderless,
Marginalia, Consult). Adding =company-mode= would introduce a
second, overlapping completion UI. Simply tell =lsp-mode= not to
look for company at all.

Add this to your =setup-lsp.el= within the =:config= block:

#+begin_src elisp
;; Prevent lsp-mode from attempting company-mode configuration
(setq lsp-completion-provider :none)
#+end_src

Then trigger CAPF registration manually by adding a hook in the
same =:config= block:

#+begin_src elisp
(add-hook 'lsp-completion-mode-hook
          (lambda ()
            (setf (alist-get 'lsp-capf
                             completion-category-defaults)
                  '((styles . (orderless))))))
#+end_src

This ensures LSP completions flow through your existing Vertico +
Orderless pipeline, and the warning disappears.

** Option B: Install Company Mode

If you /want/ the dropdown overlay experience:

1. Add =company= to your package list in =init.el=.
2. Create =setup-company.el=.
3. Change =lsp-completion-provider= back to =:capf= (company
   integrates via CAPF as a frontend).

#+begin_src elisp
;; In init.el package list, add:
;; company

;; setup-company.el
(use-package company
  :ensure t
  :hook (lsp-mode . company-mode)
  :config
  (setq company-minimum-prefix-length 1)
  (setq company-idle-delay 0.2)
  (setq company-tooltip-align-annotations t))
#+end_src

- Minibuffer Completion in Your Setup

** What It Is

The minibuffer is the single-line area at the very bottom of
your Emacs frame. When Emacs (or any package) asks you to
choose something—a file, a buffer, a command, a code
symbol—it prompts you /there/.

Your current stack makes this remarkably powerful:

| Package    | Role                                       |
|------------+--------------------------------------------|
| Vertico    | Vertical candidate list in the minibuffer  |
| Orderless  | Fuzzy, out-of-order matching of candidates |
| Marginalia | Annotations beside each candidate          |
| Consult    | Rich commands that feed the minibuffer     |

You are already using this every time you run =M-x=,
=find-file=, or =switch-to-buffer=. The same mechanism drives
code completion.

** How It Works for Code Completion

When you press =C-M-i= (bound to =completion-at-point=) in a
Clojure buffer with LSP active:

1. LSP provides a list of valid symbols at your cursor.
2. Vertico displays them vertically in the minibuffer.
3. You type fragments in any order (Orderless matches them).
4. Marginalia shows type info and namespace alongside.
5. You press =RET= to insert the chosen symbol.

** A Practical Example

Suppose you have this Clojure code:

#+begin_src clojure
(defn process-orders [db]
  (let [pending (fetch-pending-orders db)]
    (map pro|)))
#+end_src

With your cursor at =|=, press =C-M-i=. The minibuffer shows
something like:

#+begin_example
  process-orders      <f> my.namespace
  promise             <f> clojure.core
  promise?            <f> clojure.core
  proxy               <macro> clojure.core
  protocol?           <f> clojure.core
#+end_example

Now type =ord= and Orderless narrows it to =process-orders=.
Press =RET=. Done.


* Key Bindings to Practise

| Binding   | Action                                    |
|-----------+-------------------------------------------|
| =C-M-i=  | Trigger completion-at-point               |
| =C-n=    | Next candidate (in Vertico list)          |
| =C-p=    | Previous candidate                        |
| =RET=    | Accept selected candidate                 |
| =C-g=    | Abort completion                          |
| =M-.=    | Jump to definition (via LSP / lsp-ui)     |
| =M-?=    | Find references (via lsp-ui)              |
| =TAB=    | May also trigger or cycle (check binding) |

** The Mental Model

Think of it as one universal interface for /every/ selection
task in Emacs. File names, buffer names, M-x commands, git
branches (Consult), and code symbols all flow through the
same minibuffer, with the same keybindings, the same fuzzy
matching, and the same muscle memory.

That is the payoff: you learn one interaction pattern and it
works everywhere.
